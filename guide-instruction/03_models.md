# 03 — `models/` Folder Guide

## Purpose

This folder contains **pure data structures** — classes that hold data but contain  
no solving logic. Think of them as the "nouns" of the system.

No class in this folder runs an algorithm. They only store and expose data.

---

## Files

---

### `delivery_window.py` — `DeliveryWindow`

A single time slot when a customer accepts delivery.

```python
@dataclass(frozen=True)
class DeliveryWindow:
    day: int          # Day of week (1=Mon, 7=Sun)
    start_time: float # Minutes from midnight, e.g. 480.0 = 08:00
    end_time: float   # Minutes from midnight, e.g. 720.0 = 12:00
```

**Methods:**

| Method | What it does |
|---|---|
| `contains(time)` | Returns True if `time` is within `[start_time, end_time]` |
| `time_until_close(current_time)` | How many minutes until this window closes |
| `duration()` | Length of the window in minutes |

**Example:** A window for Monday 8:00–12:00 is stored as `DeliveryWindow(day=1, start_time=480.0, end_time=720.0)`.

### `customer.py` — `Customer`

Represents one delivery location (or the depot).

```python
class Customer:
    _id: str                         # Unique ID, e.g. "C001" or "DEPOT"
    _x: float                        # X coordinate in km
    _y: float                        # Y coordinate in km
    _demand: float                   # Goods needed in kg
    _service_duration: float         # Minutes to unload at this stop
    _windows: List[DeliveryWindow]   # All time windows for this customer
```

**Key methods:**

| Method | What it returns |
|---|---|
| `get_windows_for_day(day)` | Only the windows that match a specific day number |
| `get_all_windows()` | All windows across all days |
| `has_windows()` | True if the customer has at least one window |

> The depot is also a `Customer` object — it just has `demand=0`, `service_duration=0`, and no windows.

---

### `pending_queue.py` — `PendingQueue`

A set of customer IDs that have **not yet been assigned** to any route.

```python
class PendingQueue:
    _unserved: Set[str]   # e.g. {"C001", "C002", "C045", ...}
```

**Methods:**

| Method | What it does |
|---|---|
| `remove(customer_id)` | Mark this customer as served (removes from set) |
| `get_pending()` | Returns a copy of the unserved set |
| `get_all_pending()` | Returns unserved as a list |
| `is_empty()` | True if all customers have been served |
| `get_unserved_count()` | How many customers remain unserved (used for loss calculation) |

---

### `problem.py` — `Problem`

The **complete problem instance**. Created once by `DatasetLoader` and passed everywhere.

```python
class Problem:
    _depot: Customer
    _customers: Dict[str, Customer]     # All non-depot customers
    _distance_matrix: DistanceMatrix
    _travel_time_matrix: TravelTimeMatrix
    _max_days: int
    _vehicle_capacity: float
```

**Key methods:**

| Method | What it does |
|---|---|
| `get_customer(id)` | Returns a Customer by ID (also handles `"DEPOT"`) |
| `get_all_customers()` | List of all non-depot customers |
| `get_depot()` | Returns the depot Customer |
| `get_distance(from_id, to_id)` | Euclidean distance in km between two locations |
| `get_travel_time(from_id, to_id)` | Travel time in minutes between two locations |

---

### `state.py` — `State`  ⭐ Most Important Model

Represents the **solution being built**, updated as the solver works day by day.

```python
class State:
    _pending_queue: PendingQueue         # Who hasn't been served yet
    _current_day: int                    # Which day we're building right now
    _current_time: float                 # Minutes elapsed today
    _current_load: float                 # kg loaded on vehicle today
    _daily_routes: Dict[int, List[str]]  # {day: ["DEPOT", "C01", "C07", "DEPOT"]}
    _daily_arrival_times: Dict[...]      # Arrival time at each stop per day
    _daily_wait_times: Dict[...]         # Wait time at each stop per day
```

**Key methods:**

| Method | What it does |
|---|---|
| `current_route()` | Returns today's route list |
| `advance_day()` | Closes today's route (appends DEPOT), moves to next day, resets time/load |
| `commit_insertion(customer_id, index, arrival, wait)` | Inserts a customer into today's route and removes them from pending_queue |
| `get_completed_routes()` | Returns all 7 days' final routes |
| `clone()` | Deep copies the entire state (used by optimizer trials) |

**How a route looks:**
```python
_daily_routes = {
    1: ["DEPOT", "C012", "C034", "C001", "DEPOT"],
    2: ["DEPOT", "C007", "C099", "DEPOT"],
    ...
    7: ["DEPOT"]   # empty day = no deliveries
}
```

### `candidate.py` — `Candidate`

A **proposed insertion** of one customer at one position in today's route.  
Generated by `SolomonInsertion` and scored by `FeatureExtractor` + `ScoreCalculator`.

```python
@dataclass
class Candidate:
    customer_id: str          # Which customer to insert
    day: int                  # Which day
    insertion_index: int      # Where in the route to insert them
    arrival_time: float       # When the vehicle would arrive at this customer
    wait_time: float          # How long to wait before service window opens
    service_start_time: float # When service actually begins
    distance_delta: float     # Extra km added to route by this insertion

    raw_features: Dict[str, float]   # 4 raw feature values (before normalization)
    norm_features: Dict[str, float]  # 4 normalized features [0.0, 1.0]
    score: float                     # Final weighted score (lower = better)
```

---

### `vehicle.py` — `Vehicle`

A simple model of the delivery vehicle. Tracks its route, load, and timing.

```python
class Vehicle:
    id: str
    capacity: float
    current_load: float
    route: List[str]           # ["DEPOT", "C01", ...]
    arrival_times: List[float]
    wait_times: List[float]
```

> **Note:** `Vehicle` is defined but the main solver actually manages load/timing  
> directly inside `State` and `GreedySolver`. `Vehicle` exists as a supplementary model.
